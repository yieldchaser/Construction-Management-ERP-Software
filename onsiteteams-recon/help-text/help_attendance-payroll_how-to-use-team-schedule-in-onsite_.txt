Before You Start You need Admin or Senior Manager access to view the full team schedule Team members must be assigned to projects before they appear in the Schedule tab Accessing Team Schedule Click Team Schedule in the left sidebar. The module has two tabs: Schedule and Timesheet . Part 1 â€” Schedule Tab (Team Gantt View) The Schedule tab shows a Gantt-style calendar of all team members&#8217; project assignments across the company. Reading the Gantt Each row represents one team member. Coloured bars across the calendar show when they are assigned to a project. The bar spans the assignment period. Use this view to: See which team members are currently deployed and where Identify gaps in deployment (unassigned periods) Spot overlapping project assignments before adding new ones Plan manpower before starting a new project phase Filters Filter What It Does Project Show only team members assigned to a specific project. Default shows all projects. Date / Week Navigate forward or backward by week to see different time periods Part 2 â€” Timesheet Tab The Timesheet tab lists all timesheet entries logged across the company. Timesheet List Columns Column What It Shows Name Team member who logged the timesheet Project Project the work was logged against Date Date of the timesheet entry Hours Hours worked Notes Description of work done Timesheet Filters Filter Options Project Filter by project Date Set a date range Team Member Filter by a specific person Part 3 â€” Creating a New Timesheet Entry Click + New Timesheet in the top right of the Timesheet tab. Field Required What to Enter Team Member Yes Select the person from the dropdown Project Yes Select the project the work was done on Date Yes Date the work was performed Hours Yes Number of hours worked Notes No Brief description of tasks completed (e.g. &#8220;Brickwork, Ground Floor, 8 hours&#8221;) Click Save . The entry appears in the Timesheet list immediately. Tips Use the Schedule tab before assigning someone to a new project. Check whether they already have overlapping assignments during the same period. Filter the Schedule tab by project to see the full deployed team for a specific project at a glance. Log timesheet entries same-day. Entries entered days later are harder to verify against attendance records and may be disputed. Timesheet entries feed into project cost reporting. Always log against the correct project to keep project-level cost data accurate. Use the Notes field in timesheets to record what was done. Vague entries (&#8220;site work&#8221;) make cost analysis less useful than specific ones (&#8220;Plastering â€” Floors 2 and 3&#8221;).
